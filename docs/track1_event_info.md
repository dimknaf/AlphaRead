## MY PERCEPTION (clearly separated from source above)

Below this divider is *not* from the hackathon page — it's my interpretation. Treat the section above as canonical.

### Time budget reality
- Hacking starts 11:30, ends 19:30 — that's **8 hours of build time**, minus lunch and breaks → realistically **~6.5 hours** of focused build.
- Add 30 min for submission paperwork at the end (BOTH forms, screenshot, README polish).
- → **Net build window: ~6 hours.**
- Implication: scope = 1 workflow, 2-3 durable steps, 1 `DurableAgent`, simple but polished UI. Not a multi-page app. Not a complex backend. **One workflow that demonstrates the WDK headline feature really well** beats three half-finished things.

### Mubit's separate $100k pool — easy double-dip
- The Mubit-only track has its own $100k prize. The local *main* pool also gives Mubit credits to top 3.
- If the agent has any "memory" angle (learning across runs, accumulating context per user/company), bolting on Mubit (3 lines of code per their pitch) is the highest expected-value side bet of the day.
- **Recommendation:** if the chosen concept naturally has a "this gets better with more runs" angle, plan to integrate Mubit. If forced/contrived, skip it.

### Submission gotcha
- BOTH forms must be submitted separately. Easy to forget the global one (it's on a different platform — community.vercel.com — and the local pool form might feel like "done").
- Set a 19:00 alarm for "start submission paperwork."

### Judging criterion #2 reading
- "Technical execution (built with v0, deployed on Vercel)" — the parenthetical reads like an *example* of execution, not a hard requirement. Track 1 explicitly doesn't use v0 as the scaffold.
- That said: using v0 for *something* (UI components, asking it Vercel questions) likely scores well on this criterion. Plan to do at least one v0-generated UI piece even if WDK is the core.

### What "wins" looks like for Track 1
- Pick a problem where durability is genuinely necessary (not "nice to have").
- Demo it by **deliberately killing the server mid-run** and showing the workflow resumes from the last completed step. That's the WDK headline. Most submissions won't do this; it's an easy differentiator.
- Have a pretty UI (this is where v0 helps). Judges look at the screen.
- Have a 60-second pitch ready: problem → why durability → demo → impact.

### Networking / partnerships angle
- Mubit and Bright Data reps are on the judging panel. If the concept naturally uses their products, mention it explicitly in the pitch.
- Bright Data ($100 credits) is a great fit for any web-scraping step.

### Travel / logistics
- Southwark / Waterloo / Waterloo East — all 10 min walk. Pick the one with the easiest journey from home.
- Bring laptop charger (8 hours), backup phone charger, headphones. Check in at front desk on arrival.
- Catering covers the day; no need to leave the venue.
